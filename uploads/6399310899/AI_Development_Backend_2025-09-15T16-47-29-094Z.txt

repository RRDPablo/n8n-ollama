 Como programador de backend, recibiré las tareas de Scrum Master y realizaré el desarrollo del código correspondiente utilizando el lenguaje de programación especificado para cada tarea. Mantendré el código limpio y profesional, siguiendo estrictamente la definición de tareas y objetivos esperados.

Para la tarea "Desarrollo del lógica para realizar las cuatro operaciones básicas matemáticas y mostrar el resultado en la interfaz de usuario correspondiente", mi código implementará una clase Calculadora que contendrá cuatro métodos: suma(), resta(), multiplicación() y división(). Cada método recibirá dos números como parámetros, realizará la operación correspondiente y devolverá el resultado. También implementaré una función mostrarResultado(resultado) para enviar el resultado al lado frontend mediante un llamado de API.

Para la tarea "Desarrollo del código para la compatibilidad con las diferentes plataformas (Android e iOS) en la lógica y API", mi código implementará una abstracción de API para que los datos puedan ser enviados desde el lado backend a cualquier lado frontend, sea Android o iOS. Para ello crearé una clase AbstractAPI que contenga la estructura del mensaje y las reglas para construirlo correctamente. Cada implementación específica de la API se encargará de gestionar el envío del mensaje de manera adecuada según la plataforma (por ejemplo, utilizando REST en Android o NSURLSession en iOS).

Con respecto al criterio de aceptación "La calculadora debe ser capaz de realizar correctamente las cuatro operaciones básicas matemáticas", mi código cumplirá este requisito mediante la utilización de bibliotecas y métodos existentes que garanticen el cálculo correcto de estas operaciones.

Por último, para los criterios de aceptación "El usuario debe poder ingresar los números y seleccionar la operación aritmética utilizando una interfaz intuitiva y amigable al usuario" y "El resultado final de cada operación debe mostrarse en pantalla inmediatamente después de presionar el botón correspondiente", estos requisitos estarán definidos por el lado frontend, por lo que no se aplicará ninguna optimización o solución directa desde el backend.
